## 一、創作主題

### 1. 題目

**CtrlF——即時智慧尋物系統：基於視覺記憶建構之智慧環境搜尋助理（CtrlF-Memory）**

> 對著鏡頭問「我的鑰匙呢？」——畫面即時鎖定、查最後出現位置、認得「就是你那一件」，還記得「它通常放在哪」。

### 2. 實用功能描述

CtrlF 讓一台普通的 USB 攝影機成為「房間的搜尋引擎」，核心功能如下：

1. **中文搜尋、即時鎖定**：直接輸入「鑰匙」「我的錢包在哪」，內建約 90 詞辭典＋自訂同義詞自動翻譯為偵測詞彙；目標在畫面中以綠框＋半透明高亮即時標記，串流 24 fps 不受推論速度影響。
2. **最後出現位置**：每件物品的位置、時間、實物縮圖自動入庫；東西不在畫面裡時，直接告訴你它最後在哪、長什麼樣子。
3. **視覺記憶**：持續身分引擎讓「mug#1 永遠是同一個馬克杯」（跨越重啟與消失）；自動建構情節時間軸（09:12 書桌 → 14:30 沙發）與習慣統計（「通常在書桌，70% 的時間」），可回答「我最後一次看到 X 是在哪」。
4. **個人物品註冊**：上傳 3～5 張照片，系統認得「就是這一件」——我的錢包 vs 任何一個錢包；背景持續比對並更新最後行蹤，並提供「測試比對」工具驗證註冊品質。
5. **語意搜尋**：查無直接紀錄時，自動以 CLIP 跨模態比對找出視覺上最相似的目擊（搜 mobile 也能命中 cell phone 的照片）。
6. **綠色運算與隱私友善**：全程本機運算、影像不上雲，省去雲端資料中心持續收流／儲存的傳輸與運算能耗；畫面靜止不推論（動態閘控）、FP16 推論，VRAM 僅約 1 GB；紀錄 7 天自動清理，儲存不無限增長。
7. **減少電子廢棄物與重複消費**：免額外購買、免安裝任何實體防丟標籤或感測器——省下標籤鋰電池汰換所產生的電子廢棄物；協助找回既有物品可降低「找不到就重買」的消費行為，延長物品使用壽命，呼應循環經濟（Circular Economy）精神。

## 二、創意構想

### 1. 理論基礎

1. **開放詞彙物件偵測（Open-Vocabulary Detection）**：採 YOLO-World（region-text 對比預訓練），偵測類別不再受限於 COCO 80 類，而是由文字詞彙表即時定義——「pen、keys、tissue box」皆可偵測，且詞彙可於執行期動態更換。
2. **跨模態對比學習（CLIP）**：CLIP 將影像與文字映射至同一嵌入空間，本系統以此實現（a）文字→影像的語意搜尋、（b）影像→影像的個體辨識（個人物品）、（c）持續身分的外觀比對。同義詞問題由視覺語意天然解決，無需窮舉詞表。
3. **認知科學的記憶階層**：借鏡情節記憶（episodic memory）與語意記憶（semantic memory）的分層，設計「Frame（感知）→ Object（持續身分）→ Episode（情節）→ Habit（習慣）」的記憶建構鏈：感知是瞬時的，身分是持續的，情節是時空片段，習慣是統計性結論。（見【圖二】）
4. **多目標追蹤與原型學習**：IoU 貪婪匹配＋質心備援之輕量追蹤器維護短期軌跡；長期身分以 EMA（指數移動平均）原型比對——單張觀測不定生死，原型隨觀測滾動更新，並以「時空先驗」（一小時內同地點再現，比對門檻放寬）模擬人類「剛剛還在桌上的杯子大概還是同一個」的直覺。
5. **綠色人工智慧（Green AI）**：以「先過濾、後計算」的分層降載設計（頻率上限 → ROI 遮罩 → 動態觸發）取代「每幀必推」的暴力運算，在維持辨識品質的前提下大幅降低單位辨識耗能與碳足跡，呼應永續運算（Sustainable Computing）——AI 的智慧價值不應以浪費能源為代價。

### 2. 設計創新說明

1. **二階段解耦架構**：偵測器只負責「框在哪」（開放詞彙），CLIP 只負責「這是什麼、是不是同一件」（視覺語意）。兩者以佇列解耦、各自執行緒運行，任何一側可獨立替換升級。
2. **持續身分引擎（本作品核心創新）**：追蹤器 ID 只活在單次執行，本系統以「類別閘控＋EMA 原型＋時空先驗」三道保險將偵測框昇華為跨重啟穩定的物件身分，這是情節與習慣得以成立的地基；身分與目擊同步 TTL 清理，長期運行不膨脹。
3. **情節惰性建構**：情節與習慣不物化入庫，而是查詢時由目擊紀錄即時合併（7 天量級毫秒可完成）——避免非同步身分指派的排序競態，且結果永遠與事實一致。
4. **影像降載三道閘門**：推論頻率硬上限（零成本跳過）→ ROI 遮罩（背景抹黑）→ 像素差分動態觸發（靜止畫面 0 推論、定時抽樣兜底），源頭切斷 GPU 過載，也是本作品實踐綠色運算、降低單位辨識耗能的具體手段。
5. **串流與推論徹底脫鉤**：MJPEG 端點只讀狀態中樞快照並在最新影格上合成標註；N 個分頁共用一份推論與一份 JPEG 編碼——推論 5 fps 時畫面照樣 24 fps。
6. **工程規範**：依賴注入單一組裝根、模組間僅依賴 Protocol 契約、47 項自動化測試、SQLite 為事實來源＋向量索引可離線重建。

### 3. 特殊功能描述

1. **中文原生體驗**：內建約 90 詞繁中辭典（含常見簡體與英文俗名）、長詞優先子字串解析（「我的鑰匙在哪」→ keys）；聽不懂的詞主動引導使用者「教它」，同義詞即時生效。
2. **「就是這一件」個體辨識**：同類別基本門檻＋跨類別加嚴門檻的比對規則，區分「我的錢包」與「任何一個錢包」；附測試比對工具讓使用者自行驗證與調校。
3. **記憶回溯卡**：搜尋結果直接嵌入該物品的持續身分摘要——最後情節（地點／時間／停留時長）＋習慣（通常在哪、佔比），把「資料庫查詢」包裝成「像問家人一樣的回答」。
4. **執行期熱調參與自我監控**：信心門檻、敏感度、推論頻率滑桿即時生效；FPS／延遲／VRAM／儲存即時面板；前後端版本握手，錯位時主動提示。
5. **永續消費行為**：協助使用者快速找到既有物品、降低「找不到即重買」的傾向；以本機化架構取代雲端持續錄影／上傳，減少長期能源消耗與資料中心負擔，將永續實踐落實於日常尋物情境。

## 三、系統架構

### 1. 架構說明

**【圖一：系統架構圖——此處插入 figures/fig1_system_architecture.drawio 匯出圖】**

系統採 Producer–Consumer 多執行緒管線，四大模組完全解耦：

- **資料採集**：CaptureService 擷取執行緒以相機原生 FPS 讀取影格，發佈至 StateHub「最新影格信箱」（永遠覆寫最新一張，天生防止佇列堆積與延遲累積），斷線自動重連。
- **AI 核心**：InferencePipeline 推論執行緒依序通過三道降載閘門後，以 YOLO-World-M（FP16 @800）做開放詞彙偵測，IouTracker 維護物件生命週期；產出之目擊事件與嵌入任務分別交由下游佇列，推論執行緒永不寫磁碟、永不等待網路。
- **儲存與語意**：StorageWriter 批次寫入 SQLite（WAL）；EmbeddingWorker 對裁切計算 CLIP 影像嵌入寫入內嵌向量索引（正規化矩陣內積，亞毫秒精確搜尋、零部署），同時完成持續身分指派與個人物品比對。
- **展示層**：FastAPI 非同步後端只讀 StateHub 快照與儲存層（MJPEG／WebSocket／REST）；Streamlit 前端為純消費者，零業務邏輯。

依賴注入之組裝根集中於單一檔案，模組間僅依賴 Protocol 契約——偵測器、嵌入器、向量庫皆可單獨替換（如將來換 YOLOE、上 Qdrant）。

### 2. 「人機介面設計」（UI）與「使用者體驗」（UX）設計

**【圖三：UI 版面配置示意——此處插入 figures/fig3_ui_layout.drawio 匯出圖】**

- **版面**：左側欄承載「系統監控／偵測調校／常駐追蹤／同義詞辭典」等低頻設定；主畫面以「搜尋框＋快速標籤」為第一視覺焦點，下分三分頁：即時尋物、記憶時間軸、物品註冊管理。
- **畫面永不中斷**：影像走 iframe 內 MJPEG（自動重連＋骨架屏動畫），效能資訊走 WebSocket HUD——兩者皆不觸發頁面重繪；搜尋結果、我的物品等區塊以 fragment 局部更新（1.5～3 秒），互動不會讓畫面閃爍。
- **白話文案與引導**：三步驟新手引導、「就在畫面裡！」「目前不在畫面中，持續掃描…」等口語回饋；未收錄中文詞主動引導補辭典；註冊流程附品質檢測（測試比對），失敗時給出具體建議（補拍角度、調整門檻）。
- **一致的設計系統**：卡片／徽章／步驟元件統一樣式，縮圖統一規格；狀態徽章（●運作中・CUDA・版本）常駐右上；危險操作（刪除）與長時操作（上傳計算特徵）皆有明確回饋。
