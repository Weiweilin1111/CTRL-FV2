# 2026 年全國大專校院智慧創新暨跨域整合創作競賽

# 系統需求書

---

## 1. 系統名稱

**CtrlF——即時智慧尋物系統**
（研究定位：CtrlF-Memory——基於視覺記憶建構之智慧環境搜尋助理）

系統簡稱：**CF**

## 2. 系統目的與範圍

**目的：**
「東西又找不到了」是每個家庭的日常：鑰匙、錢包、遙控器、藥盒——市售定位標籤（如 AirTag）只能找「有掛牌」的物品，監視器回放則需人工逐格翻找。本系統讓一台普通的 USB 攝影機成為「房間的搜尋引擎」：使用者對介面輸入「我的鑰匙呢？」，系統即時在畫面中以綠框鎖定目標；若目標不在畫面內，則回答它「最後一次出現的位置、時間與實物照片」，並進一步以視覺記憶回答「這件物品通常放在哪」。

系統以開放詞彙偵測（YOLO-World）與跨模態視覺語意（CLIP）二階段架構實現「零標籤、免訓練、講中文」的尋物體驗，並以「Frame（影格）→ Object（持續身分）→ Episode（情節）→ Habit（習慣）」的記憶建構鏈，讓系統從「即時偵測工具」升級為「記得東西習慣放哪」的環境記憶助理。

本系統亦以永續設計為核心考量：全程本機運算取代雲端持續錄影與傳輸，以動態閘控降低推論耗能；免除實體防丟標籤（省下電池與電子廢棄物），並藉由協助找回既有物品，降低「找不到就重買」之消費行為，延長物品使用壽命。

**範圍：**

| 項目 | 說明 |
|---|---|
| 包含 | 單支 USB 攝影機之室內固定場景；即時偵測、追蹤與串流；中文／英文搜尋；最後出現位置查詢；跨模態語意搜尋；視覺記憶（持續身分／情節／習慣）；個人物品照片註冊與辨識；執行期參數調校與系統監控 |
| 不包含 | 多攝影機協同（保留擴充介面）；雲端服務（隱私設計上刻意排除）；人臉辨識或人物追蹤 |
| 運行環境 | Windows PC＋NVIDIA GPU（實測 RTX 5070 Ti Laptop，VRAM 需求約 1 GB）；所有運算與儲存皆在本機完成 |

## 3. 系統非功能需求

| 非功能需求編號 | 非功能需求描述 |
|---|---|
| CF-NF-001 | **即時性**：影像串流維持 24 FPS 以上，且與 AI 推論頻率完全脫鉤（推論 5 FPS 時畫面依然流暢）；單幀推論延遲 ≤ 100 ms（實測 43 ms／幀）。 |
| CF-NF-002 | **資源效率／綠色運算**：GPU 記憶體佔用 ≤ 1.5 GB（實測 0.7～1.1 GB）；畫面靜止時不進行推論（像素差分動態閘控），降低單位辨識耗能；多瀏覽器分頁同時觀看永遠只有一份推論與一份 JPEG 編碼。 |
| CF-NF-003 | **隱私性**：所有影像於本機運算與儲存，不上傳任何雲端服務；照片註冊僅保留縮圖與視覺特徵向量，不保存原圖。 |
| CF-NF-004 | **儲存控管**：目擊紀錄保留 7 天（可設定），逾期自動自 SQLite 與向量索引同步清除；過期之物件身分一併清理；儲存增長約 1 MB／15 分鐘且有上限。 |
| CF-NF-005 | **易用性**：介面以繁體中文呈現、文案白話（非資訊背景使用者可讀），提供三步驟新手引導；中文查無對應詞時主動引導使用者補充同義詞。 |
| CF-NF-006 | **可靠性**：相機斷線自動重連；單幀推論失敗不中斷管線；CLIP 元件不可用時自動降級為字串比對（Null Object 模式），系統不崩潰。 |
| CF-NF-007 | **資料韌性**：SQLite 為唯一事實來源；向量索引損毀或更換嵌入模型時，可由目擊縮圖以離線工具完整重建。 |
| CF-NF-008 | **可維護性／可擴充性**：模組間僅依賴介面契約（依賴注入＋Protocol），偵測器、嵌入器、向量庫皆可獨立替換；具 47 項自動化測試（後端 33 項端點回歸＋UI 6 項真實互動＋記憶層 8 項離線單元）。 |
| CF-NF-009 | **環境永續**：以本機化運算取代雲端持續錄影／傳輸，降低資料中心能耗負擔；免除實體防丟標籤及其電池汰換之電子廢棄物；協助找回既有物品，降低因遺失而重複購買之消費行為，延長物品使用壽命。 |

## 4. 系統功能需求

| 功能需求編號 | 功能需求描述 |
|---|---|
| CF-F-001 | **即時影像串流與目標鎖定**：以 MJPEG 串流即時顯示畫面；搜尋命中之目標以綠框＋半透明高亮＋標籤晶片即時標記，其餘物件可選擇性以淡色細框顯示。 |
| CF-F-002 | **中文／英文搜尋**：輸入中文（如「鑰匙」「我的錢包在哪」）經「自訂同義詞 → 內建辭典（約 90 詞）→ 長詞優先子字串比對」解析為偵測詞彙；未收錄之中文回報未解析並引導補充辭典。 |
| CF-F-003 | **開放詞彙偵測與追蹤**：以 YOLO-World 依動態詞彙表（基礎詞彙 ∪ 常駐標籤 ∪ 當前查詢 ∪ 區域錨點）偵測任意名詞描述之物件；IoU 貪婪匹配＋質心備援之多目標追蹤器維護物件生命週期，過濾單幀誤檢。 |
| CF-F-004 | **區域定位**：以區域錨點（書桌、床、沙發、層架…）判定物件所在位置，取面積最小之包含區域作為語意最精準的容器。 |
| CF-F-005 | **最後出現位置**：每件物品之位置、起訖時間與實物縮圖自動入庫；查詢時依時間列出最近目擊紀錄卡片。 |
| CF-F-006 | **跨模態語意搜尋**：查無直接紀錄時，將查詢文字編碼為 CLIP 向量，與歷史目擊之影像向量做餘弦比對，回傳視覺上最相似的紀錄（搜 mobile 可命中 cell phone 的目擊照片）。 |
| CF-F-007 | **視覺記憶——持續身分**：以 CLIP 嵌入為每次目擊指派跨重啟穩定的物件身分（類別閘控＋EMA 原型＋時空先驗），「mug#1 永遠是同一個馬克杯」。 |
| CF-F-008 | **視覺記憶——情節與習慣**：同一身分於同一地點之連續目擊自動合併為情節（Episode）；依停留時間加權統計習慣分佈（「通常在書桌，70% 的時間」）；支援「我最後一次看到 X 是在哪」之回溯查詢（吃中文）。 |
| CF-F-009 | **個人物品註冊**：上傳 3～5 張參考照註冊個人物品；系統對每個即時裁切於背景自動比對（同類別門檻＋跨類別加嚴），命中即更新該物品之最後行蹤（位置／時間／相似度／縮圖）。 |
| CF-F-010 | **註冊管理與測試比對**：參考照逐張檢視／刪除／補充、物品改名／刪除；提供測試比對工具——上傳一張現場照，立即回報相似度與能否被認出。 |
| CF-F-011 | **詞彙與同義詞管理**：常駐追蹤標籤（加入後持續記錄行蹤）、近期搜尋標籤（一鍵重搜）、中文同義詞新增／刪除即時生效。 |
| CF-F-012 | **執行期調校與監控**：偵測信心門檻、畫面變化敏感度、推論頻率上限以滑桿即時調整（不需重啟）；即時監控串流／推論 FPS、延遲、VRAM、儲存用量，並具前後端版本一致性檢查。 |

### 一般性的系統功能操作使用案例（Use case, UC）之劇本（Scenario）描述如下

**使用案例編號：CF-UC001　使用案例名稱：中文搜尋即時尋物**

| 系統反應動作 | 使用者操作動作 |
|---|---|
| | a. 於搜尋框輸入「鑰匙」並按下 Enter（或點選快速標籤） |
| b. 依「自訂同義詞 → 內建辭典 → 子字串比對」解析為 keys，顯示翻譯提示「已自動翻譯：鑰匙 → keys」，並將該詞即時加入偵測詞彙表與近期標籤 | |
| c. 目標出現於畫面時，即時以綠框＋半透明高亮鎖定，顯示「就在畫面裡！鎖定 N 個目標」與所在區域、信心值 | |
| | d. 依畫面綠框位置找到物品 |

**使用案例編號：CF-UC002　使用案例名稱：查詢最後出現位置（含記憶回溯與語意搜尋）**

| 系統反應動作 | 使用者操作動作 |
|---|---|
| | a. 搜尋一件目前不在畫面中的物品（如「錢包」） |
| b. 顯示「目前不在畫面中，持續掃描…」；同時列出（1）記憶回溯卡：該物品持續身分之最後情節（地點／時間／停留時長）與習慣（通常在書桌，70%）（2）最後出現位置卡片：時間、區域與實物縮圖 | |
| c. 若查無任何直接紀錄，自動改以 CLIP 跨模態語意搜尋，列出「AI 找到最相似的」目擊紀錄與相似度 | |
| | d. 依最後位置／習慣地點前往尋找；或點選系統建議的相近詞改搜 |

**使用案例編號：CF-UC003　使用案例名稱：註冊個人物品（認得「就是這一件」）**

| 系統反應動作 | 使用者操作動作 |
|---|---|
| | a. 切換至「物品註冊管理」分頁，填寫名稱（可中文）與英文類別（如 wallet），上傳 3～5 張不同角度特寫照 |
| b. 校驗欄位與圖片格式；於後端計算各照片之 CLIP 影像嵌入並寫入個人物品向量索引，僅保留縮圖不存原圖；註冊之類別立即加入偵測詞彙 | |
| c. 顯示註冊成功訊息與參考照張數；此後每個即時裁切皆於背景與該物品比對，命中即更新「最後行蹤」 | |
| | d. 於「我的物品」區檢視最後行蹤，或點「找它」一鍵搜尋 |

**使用案例編號：CF-UC004　使用案例名稱：測試比對（註冊品質檢測）**

| 系統反應動作 | 使用者操作動作 |
|---|---|
| | a. 於該物品管理面板上傳一張現場照，點選「開始測試」 |
| b. 計算測試照與該物品全部參考照之最高相似度，以進度條顯示相似度與判定門檻 | |
| c. 相似度達門檻顯示「認得出來」；未達門檻顯示「認不出來」並建議補拍更近、更多角度之參考照或調整門檻 | |
| | d. 依建議補充參考照後重新測試 |

**使用案例編號：CF-UC005　使用案例名稱：查看記憶時間軸（情節與習慣）**

| 系統反應動作 | 使用者操作動作 |
|---|---|
| | a. 切換至「記憶時間軸」分頁，自清單選擇一件物品（如 mug#1） |
| b. 顯示該身分之首次／最後出現資訊；以進度條呈現習慣分佈（依停留時間加權之地點百分比） | |
| c. 以卡片列出情節時間軸：每段情節之地點、起訖時間與停留分鐘數（09:12 書桌 → 14:30 沙發） | |
| | d. 依習慣分佈判斷物品最可能的存放處 |

**使用案例編號：CF-UC006　使用案例名稱：新增中文同義詞**

| 系統反應動作 | 使用者操作動作 |
|---|---|
| | a. 搜尋「充電頭」 |
| b. 辭典查無對應，顯示「還聽不懂這個詞」警告並引導至側欄辭典 | |
| | c. 於側欄「中文同義詞辭典」輸入 充電頭 → charger，點選新增 |
| d. 寫入同義詞表並即時重載辭典；此後「充電頭」解析為 charger，全系統（搜尋／語意／回溯）即刻生效 | |

**使用案例編號：CF-UC007　使用案例名稱：執行期參數調校**

| 系統反應動作 | 使用者操作動作 |
|---|---|
| | a. 於側欄拖動「偵測信心門檻」「畫面變化敏感度」「AI 分析頻率上限」滑桿，或切換「顯示所有偵測物件」 |
| b. 參數經範圍夾制後即時套用至推論管線（不需重啟）；標註層立即以新設定重繪 | |
| c. 側欄監控面板同步更新推論 FPS、延遲與畫面動態比率，供使用者確認調整效果 | |
| | d. 依監控數據微調至滿意的靈敏度／省電平衡 |

**使用案例編號：CF-UC008　使用案例名稱：常駐追蹤與行蹤紀錄**

| 系統反應動作 | 使用者操作動作 |
|---|---|
| | a. 於側欄「常駐追蹤」輸入 medicine bottle 並加入 |
| b. 常駐標籤立即併入偵測詞彙；該物品此後持續記錄出現位置（不特別框出），並參與視覺記憶建構 | |
| | c. 需要時直接搜尋該物品 |
| d. 即時鎖定或回覆最後出現位置、情節與習慣 | |
